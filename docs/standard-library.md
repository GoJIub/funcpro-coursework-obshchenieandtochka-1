# Standard Library

## Арифметика

| Функция | Описание | Пример |
|---|---|---|
| `+` | сложение | `(+ 1 2)` → `3` |
| `-` | вычитание | `(- 5 3)` → `2` |
| `*` | умножение | `(* 3 4)` → `12` |
| `/` | целочисленное деление | `(/ 10 2)` → `5` |
| `=` | равенство чисел и булевых | `(= 1 1)` → `true` |
| `<` | меньше | `(< 1 2)` → `true` |
| `>` | больше | `(> 2 1)` → `true` |

Ошибки: `TypeMismatch` при передаче не-числа, `DivisionByZero` при делении на 0.

## Списки

| Функция | Сигнатура | Описание |
|---|---|---|
| `list` | `any... → List` | создать список из аргументов |
| `head` | `List → Value` | первый элемент |
| `tail` | `List → List` | список без первого элемента |
| `cons` | `Value List → List` | добавить элемент в начало |
| `empty?` | `List → Bool` | проверить пустоту |

```lisp
(list 1 2 3)          ; → (list 1 2 3)
(head (list 1 2 3))   ; → 1
(tail (list 1 2 3))   ; → (list 2 3)
(cons 0 (list 1 2))   ; → (list 0 1 2)
(empty? (list))       ; → true
```

Ошибки: `TypeMismatch` при передаче не-списка, `OtherEvalError` при
вызове `head`/`tail` на пустом списке.

## Функции высшего порядка

| Функция | Сигнатура | Описание |
|---|---|---|
| `map` | `(Value → Value) List → List` | применить функцию к каждому элементу |
| `filter` | `(Value → Bool) List → List` | оставить элементы где предикат true |
| `fold` | `(Value Value → Value) Value List → Value` | свёртка списка слева |

```lisp
(map (lambda (x) (* x 2)) (list 1 2 3))      ; → (list 2 4 6)
(filter (lambda (x) (> x 2)) (list 1 2 3 4)) ; → (list 3 4)
(fold + 0 (list 1 2 3))                       ; → 6
```

## Maybe

`Maybe` представляет значение которое может отсутствовать.

| Функция | Сигнатура | Описание |
|---|---|---|
| `just` | `Value → Maybe` | обернуть значение |
| `nothing` | `→ Maybe` | пустое значение |
| `fmap` | `(Value → Value) Maybe → Maybe` | применить функцию внутри Maybe |
| `bind` | `Maybe (Value → Maybe) → Maybe` | монадическая цепочка |

```lisp
(just 5)                              ; → (just 5)
(nothing)                             ; → nothing
(fmap (lambda (x) (* x 2)) (just 5)) ; → (just 10)
(fmap (lambda (x) (* x 2)) (nothing)); → nothing
(bind (just 4)
  (lambda (x) (just (* x 2))))       ; → (just 8)
```

## Ленивость

Язык строгий по умолчанию. Ленивость реализуется явно через
специальные формы `delay` и `force`.

| Форма | Описание |
|---|---|
| `delay` | отложить вычисление выражения |
| `force` | вычислить отложенное значение |

```lisp
(let t (delay (+ 1 2))
  (force t))
; → 3, выражение (+ 1 2) вычислено только при force
```

Мемоизация — повторный `force` возвращает кэшированный результат,
выражение не пересчитывается:

```lisp
(let t (delay (+ 1 2))
  (let a (force t)
    (let b (force t)
      b)))
; → 3, (+ 1 2) вычислено только один раз
```

Лексическое окружение захватывается в момент `delay`:

```lisp
(let x 10
  (let t (delay x)
    (let x 20
      (force t))))
; → 10, не 20
```
